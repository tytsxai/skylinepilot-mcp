# SkylinePilot MCP 品牌与风格指南（提案）

> 目标：在功能不变前提下，统一“名字、视觉、语言、接口风格”，让产品从“能用”升级为“专业可持续”。

---

## 1. 品牌命名

- 推荐主名：`SkylinePilot MCP`
- 推荐中文名：`电报自动化控制台`
- 一句话描述：`面向生产环境的 Telegram 自动化与多账号运营平台`

命名规则：
1. 对外产品名统一 `SkylinePilot MCP`
2. 代码包名统一 `skylinepilot`
3. 模块命名使用英文小写下划线（如 `message_service.py`）
4. 用户可见文案使用中文、术语统一（账号/代理/任务/模板）

---

## 2. 视觉风格

### 2.1 颜色 Token

| Token | 颜色值 | 用途 |
|---|---|---|
| `--color-primary` | `#4F46E5` | 主按钮、主链接 |
| `--color-success` | `#16A34A` | 成功态 |
| `--color-warning` | `#D97706` | 警告态 |
| `--color-danger` | `#DC2626` | 风险/删除 |
| `--color-bg` | `#0F172A` | 主背景 |
| `--color-card` | `#1E293B` | 卡片容器 |

### 2.2 字体层级

- `h1`: 28px / semibold
- `h2`: 22px / semibold
- `h3`: 18px / medium
- `body`: 14px / regular
- `caption`: 12px / regular

### 2.3 交互语气

- 成功：清晰、简短、可确认（例：“任务已创建，将在 20:30 执行”）
- 失败：说明原因 + 下一步建议（例：“代理连接失败，请检查 IP 与端口”）

---

## 3. 接口风格

统一响应结构：

```json
{
  "ok": true,
  "data": {},
  "error": null
}
```

错误响应：

```json
{
  "ok": false,
  "data": null,
  "error": {
    "code": "PROXY_CONNECT_TIMEOUT",
    "message": "代理连接超时"
  }
}
```

---

## 4. 文案规范

1. 避免同义词混用：  
   - 统一用“账号”，不用“账户/用户号”混写  
   - 统一用“任务”，不用“计划/调度”混写（技术文档可用调度）
2. 操作按钮动词统一：`创建 / 保存 / 测试 / 删除 / 导出 / 刷新`
3. 时间格式统一：`YYYY-MM-DD HH:mm:ss`（24 小时制）

---

## 5. 落地优先级

1. 先统一命名与文案（低风险，高收益）
2. 再统一 API 返回结构（兼容层过渡）
3. 最后改造前端视觉与组件（分模块替换）
